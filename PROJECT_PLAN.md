# TruthScout - Project Plan (Hackathon MVP)

## 1. 核心理念 (Core Concept)
一个通过众包验证的社交媒体内容可信度评估平台。用户通过滑动交互（真 / 部分真实 / 假）对帖子进行投票，系统根据用户的信誉分加权计算帖子的最终可信度得分。

## 2. 核心算法 (Algorithms)

### A. 用户信誉分 (User Accuracy Score)
- **范围**: -0.8 到 +4.0
- **初始值**: 0 (权重 = 1.0)
- **调整规则**:
  - 投票与最终社区共识一致 → +0.1
  - 投票与最终社区共识不一致 → -0.2 (惩罚力度更强)
- **权重公式**: `用户投票权重 = 1 + 信誉分`

### B. 帖子可信度得分 (Post Trust Score)
- **公式**: `(所有投票权重 × 投票方向(1/0) 的总和) / (所有投票权重总和)`
- **显示方式**: 0-20% (红), 21-40% (橙), 41-60% (灰/不确定), 61-80% (浅绿), 81-100% (深绿)

### C. 时间衰减 (Vote Decay)
- 为了打破“先入为主”的认知偏差，旧的投票权重会随时间的推移而略有降低。
- **实现策略**: 24小时前的投票，其权重自动乘以 0.9。

### D. 三向滑动交互 (3-Way Swipe)
- 用户交互方式：左滑（假 / False）、下滑（部分真实 / Partial）、右滑（真 / True）。
- **逻辑逻辑**：
  - 若最终共识为“部分真实”，则投“部分真实”的用户信誉分增加，投“真”或“假”的用户信誉分减少。

## 3. 系统分类 (Content Classification)
- **分类器 (Classifier)**: 通过 Vultr LLM API 判断帖子是“事实 (FACTUAL)”还是“观点 (OPINION)”。
  - 事实 → 触发投票系统 (影响信誉分)。
  - 观点 → 显示为讨论帖 (显示同意/不同意比例，不影响信誉分)。
- **提示词 (Prompt)**: *"Classify this post into EXACTLY one category: FACTUAL or OPINION."*

## 4. 初步架构 (MVP Scope)
1.  **数据层**: Supabase (Users, Posts, Votes 表)。
2.  **交互层**: Next.js + Shadcn UI (Feed流, 滑动卡片, 实时得分更新)。
3.  **AI 层**: Vultr (帖子分类), Gradium (语音播报得分)。
